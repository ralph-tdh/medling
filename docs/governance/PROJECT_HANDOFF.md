## 3. Git gotchas (learned the hard way)
- Pushing any `lessons/*.json` change triggers the **audio workflow**, which auto-commits MP3s back to `main`.
  Always `git pull --rebase origin main` before the next push or it rejects.
- PowerShell piping a token into `gh auth login --with-token` **corrupts it** (UTF-16). Read tokens from a file,
  or just call the GitHub REST API directly with the token in an `Authorization: token …` header.
- Refresh PATH in each PowerShell call before `gh`/`git`:
  `$env:Path = [Environment]::GetEnvironmentVariable("Path","Machine") + ";" + [Environment]::GetEnvironmentVariable("Path","User")`
- Commit messages: use multiple `-m` flags (here-strings break when the closing `'@` isn't alone at column 0).

## 4. Architecture (locked — D13/D19/D20/D26)
**Vanilla HTML/JS + JSON lessons. No framework** (D26 ratified vanilla). Shell renders, content is data.
- `app/index.html` — lesson picker (reads `lessons/index.json` + `roadmap.json`) and player host.
- `app/engine.js` — the whole player (welcome → situations → branching dialogue → quiz → flashcards →
  revision quiz). Split a module out only when a file passes ~1000 lines (D20).
- `app/engine/*.js` — optional modules (dialogue, morphology, notebook/IndexedDB, fsrs, auth) — engine guards
  for their absence so a standalone build still plays.
- `lessons/*.json` — one lesson per file (PB1–4, 1a-01..10). `index.json` = picker list. `roadmap.json` = coming-soon.
- `audio/<id>/*.mp3` — generated by CI. `brand/` — Atelier tokens/logos. `supabase/` — backend (mocked until Gate 5/D30).
- `scripts/`: `validate_lessons.py` (schema), `generate_audio_v2.py` (edge-tts, CI), `build.js` (one standalone .html).

## 5. Conventions / non-negotiables
- **Lesson schema = FLAT** (D15): `pq_en`, `opts`, `tip_en` on situation objects; never nested `practice_q`.
  Exactly one `ok:true` per opts; `gl` on every option; IPA without slashes; bilingual EN/VI everywhere (D10).
- **US English primary** (D33): vocab `ipa` = US, optional `ipa_gb` = GB (only where it differs); spelling-different
  words add `en_gb` (US in `en`). A global US·GB toggle (default US, `localStorage['medling.accent']`) swaps the
  *displayed* vocab IPA/spelling via `vIpa()`/`vEn()`; **audio stays US** in both modes. Prose is US.
- **Audio** = edge-tts free pool ONLY (currently en-US/CA voices, no `*MultilingualNeural` — D22). Clip naming
  `{id}_s{si+1}_pt|p{i}|v{i}.mp3`, `{id}_q{i}.mp3`, keyed by situation **index+1** (matches engine `clipPath`).
- **Content = faithful to source textbooks** (the moat). Numbers/definitions/morphemes checked vs source.
- **The 5 gates:** agents run 1 (source-fidelity), 3 (UAT), 4 (analytics). **Only Ralph** passes 2 (sign lessons)
  and 5 (anything irreversible/published: deploy, schema, payment). Both are currently ✅ for PB + 1A.
- **Decisions** get immutable D-numbers in `decision_log.md` (never renumber); agents propose, Ralph ratifies.
- **File discipline:** content files ≤~150 lines / 8k chars at unit boundaries; strategy docs ≤300–400; INDEX small.
- Reports/strategy to Ralph in **Vietnamese** (casual); code/commits/schema in **English**.

## 6. How to (common tasks)
- **Run locally:** static server at repo root, e.g. `python -m http.server 8081` → open
  `http://localhost:8081/app/index.html?lesson=1a-01`. (Preview tool config: `medling (official build)` on 8081.)
- **Validate lessons:** `python scripts/validate_lessons.py` → expect `PASS — N files, 0 issues`.
- **Add a free lesson:** write `lessons/<id>.json` (flat schema; US-primary IPA) → validate → push.
  CI auto-generates its MP3s and rebuilds `index.json`; picker shows it automatically. No code changes needed.
- **Generate audio manually:** `python scripts/generate_audio_v2.py` (interactive) / `--ci` (skip-existing, CI mode).
- **Standalone single-file build:** `node scripts/build.js lessons/<id>.json out.html` (runs from file://).
- **Deploy:** already live; pushes to `main` auto-redeploy Pages. Full runbook + Supabase steps: `docs/DEPLOY.md`.
