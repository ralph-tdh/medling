# MedLing Audio Pipeline

Audio is generated by CI from lesson JSON — never authored by hand.

## Contract (do not break)
- Source: `lessons/*.json` (auto-discovered). Output: `audio/<lesson_id>/...`. Manifest:
  `audio/manifest.json`.
- Clip naming: `{id}_s{sid}_pt.mp3` (patient), `{id}_s{sid}_p{i}.mp3` (phrase),
  `{id}_s{sid}_v{i}.mp3` (vocab), `{id}_q{i}.mp3` (quiz). `engine.js` `clipPath()` and the Python
  generator MUST agree on this.
- Voice: seeded by `section_id` via `random.Random(hash(section_id) & 0xFFFFFFFF)` (stable re-runs).
- **FREE voice pool ONLY** (10 voices) — NEVER `*MultilingualNeural` (Azure paid):
  en-GB-SoniaNeural, en-GB-RyanNeural, en-GB-LibbyNeural, en-CA-LiamNeural, en-IE-EmilyNeural,
  en-US-MichelleNeural, en-US-GuyNeural, en-US-JennyNeural, en-AU-NatashaNeural, en-AU-WilliamNeural.

## New lesson workflow
1. Write `lessons/<id>.json` (audio NOT enabled yet).
2. Push to GitHub → workflow `.github/workflows/generate_audio.yml` runs
   `python scripts/generate_audio_v2.py --ci` (non-interactive, overwrite=false, no push prompt) →
   commits new MP3s + rebuilt `index.json` with `[skip ci]`.
3. Verify clips exist under `audio/<id>/`, then set `AUDIO_ENABLED = true` in the engine for that
   lesson and push.

## Paths in this repo (official layout)
Script lives at `scripts/generate_audio_v2.py`; the workflow calls it by that path. `engine.js`
reads clips from `../audio/...` (player is in `app/`, audio at repo root). If you move files, update
both the workflow `paths:`/`run:` and `clipPath()` together.

## Troubleshooting "no sound"
- Clip 404 → naming mismatch or CI hasn't run yet → engine falls back to browser TTS.
- All silent → `AUDIO_ENABLED` still false, or paid voice rejected by edge-tts.
- edge-tts is unofficial (D22) — if it breaks upstream, MP3s already generated still play; plan
  licensed-TTS fallback for premium/scale.
